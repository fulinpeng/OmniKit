# Excel转Word自动化程序（Word版本）

将Excel表格中的数据自动填充到Word模板，并为每一行数据生成独立的Word文档。

## 功能说明

- 读取Excel文件中的数据（`Sheet1` 工作表）
- 将数据填充到Word模板的 `{0}` 占位符位置
- 为每个员工生成独立的Word文档，文件名为 `A列_F列.docx`

## 文件结构

```
excel2word/
├── input/              # 输入文件夹
│   ├── a.xlsx         # Excel数据源
│   └── b.docx         # Word模板
├── output/            # 输出文件夹（生成的Word文档）
├── index_word.js      # 主程序（Word版本）
├── package_word.json  # 依赖配置（Word版本）
└── README_WORD.md     # 说明文档
```

## 字段映射

Word模板中的占位符使用 `{0}` 格式，程序会按以下顺序依次替换：

| 替换顺序 | Excel列 | 说明 |
|---------|---------|------|
| 第1个 `{0}` | E列 | 第1个数据 |
| 第2个 `{0}` | B列 | 第2个数据 |
| 第3个 `{0}` | A列 | 第3个数据 |
| 第4个 `{0}` | D列 | 第4个数据 |
| 第5个 `{0}` | C列 | 第5个数据 |
| 第6个 `{0}` | E列 | 第6个数据 |

## 安装步骤

1. 确保已安装 Node.js（建议 v14 或更高版本）

2. 安装依赖：
```bash
npm install
```

## 使用方法

1. 将Excel数据文件放到 `input/a.xlsx`
2. 将Word模板文件放到 `input/b.docx`
3. 确保Word模板中的占位符格式为 `{0}`（如：`Payment Date: {0}`）
4. 运行程序：
```bash
npm start
```

5. 生成的Word文档将保存在 `output/` 文件夹中

## 注意事项

- Excel文件必须包含名为 `Sheet1` 的工作表
- Word模板中的占位符格式必须是 `{0}`
- 生成的文件名为 `A列_F列.docx`（如：`COO_23078862.docx`）
- 程序会自动识别有数据的行（B列不为空）
- 如果某行数据的员工姓名为空，将跳过该行

## 依赖库

- `xlsx` - 读取Excel文件
- `docxtemplater` - 处理Word模板
- `pizzip` - 处理docx文件的zip结构

## 特色功能

- ✅ **中文编码支持** - 正确处理中文字符
- ✅ **控件保护** - 保护Word模板中的复选框等控件
- ✅ **自动数据识别** - 智能识别有效数据行
- ✅ **日期格式转换** - 自动转换Excel日期格式
- ✅ **错误处理** - 完善的错误处理和日志输出

## 问题排查

如果程序运行出错，请检查：

1. Excel文件路径和工作表名称是否正确
2. Word模板文件是否存在
3. Word模板中的占位符格式是否为 `{0}`
4. Excel数据列是否对应正确（E、B、A、D、C、E列）
5. A列和F列数据是否包含特殊字符
